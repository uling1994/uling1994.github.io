正则表达式的一千零一夜。

发表于 2017-01-14

Regular expression

  1. 语法；
  2. 在IDE使用正则处理复杂的字符串查找替换功能；
  3. Js中使用正则处理字符串；

大白话就是：按规则匹配符合条件的字符串，这就是正则表达式。

有很多的符号都有自己特殊的含义，for example:

  1. .点表示任意字符；
  2. *表示全部字符；
  3. [ ]表示或者；
  4. ^表示开始（也可以取反），$表示结束；
  5. ( )表示分组；
  6. | 或，很好理解；
  7. $ 反向引用，代替每个分组；

此处推荐一个在线匹配正则表达式可视化工具.它会以流程图的形式表现出正则表达式，有助于理解，也可以clone到本地，通过本地端口打开，速度奇快~

上面说了常用的符号，下面是字母的特殊含义，for example：

  1. \b ……\b匹配字符；
  2. \w 比配单词；
  3. g: 全局匹配，常放到\b；
  4. i:ignore case 忽略大小写；
  5. m:multiple lines 多行搜索；

元字符+原意文本字符=正则表达式

元字符:

  1. \n 换行；
  2. \r 回车；
  3. \o 空格；
  4. \f 换页；
  5. \cX 与X对应的控制字符；

范围类：[a-zA-Z]从小写a到大写Z。（常用于邮箱匹配等交互中）
预定义类：

  1. \d=[0-9]
  2. \D=[^0-9] 大小写表示取反；
  3. \w=[a-zA-Z_0-9]字母到数字都归劳资管；

单词边界：\b同理大写\B为取反：非单词边界。
量词：

  1. ？出现0-1次；
  2. +出现大于等于一次；
  3. {n}出现n次；
  4. {n,}至少出现了n次；

前瞻与后顾

前瞻：向前检查是否符合断言（尾部）；
后顾：相反（头部）*（Js并不支持，不考虑）；
正向前瞻：exp（？=assert）；
负向前瞻：exp（？！assert）；

RegExp.prototype.exec(str)

更新全局Reg对象并反映匹配结果：null与数组；
index是被匹配的第一个字符；
input输出这个string；

stringI.prototype.exec(str)

  1. search搜索很多项时返回第一项，没有则返回-1；
  2. 不支持全局匹配；
  3. 搜索指定字符串中的子字符串或与正则表达式相匹配的子字符串；

match：找一个或多个与其匹配文本。

  1. 全局，全部都匹配；
  2. 非全局，仅匹配第一个子字符串；

.split(reg)

就正如字面所写，就是把字符串中的某类字符删除

.replace(“1”,2)

  • （str，replaceStr）
  • （reg，replaceStr）
  • （reg，function）
  1. 匹配字符串；
  2. 正则分组内容，没有分组则没有参数；
  3. 匹配项中的index；
  4. 原字符串
