作为一个小白的我，曾经一度认为weex和微信小程序是有一腿的（对不起，鞠躬；）
我今天看了 @余博伦 的文章之后，很赞同一句话，那就是微信小程序离把app玩出花来还有很长的一段时间，但是很想看看如果微信这大事情搞成了，会是一副什么样子？（这是后话）

今天有个小目标：微信小程序DIY （Day One）

前面的注册下载请自行（baidu/Google） 链接

你会在根文件看到三个文件很赤裸的出现在你面前

  • app.js 他是微信小程序的逻辑；
  • app.json 他是小程序的公共设置；
  • app.wxss 他是小程序的公共样式表；

但是一个小程序页面并不是光由这三个文件组成的

  • 还有一个wxml的页面结构

注意：为了方便开发者减少配置项，我们规定描述页面的这四个文件必须具有相同的路径与文件名。

app.json

app.json是对小程序进行全局配置，从而决定页面文件的路径，窗口表现，设置网络超时，等等；

  • pages 设置页面路径，【路径+文件名】，文件名不需要加后缀，框架会自动寻找路径整合；
  • window 设置默认页面的窗口表现，例如：状态栏，导航条（navigation），标题，等等颜色背景色；
  • tabBar 设置底层（顶部）tab的表现，它是一个数组，最少两个，最多五个，按顺序排序；
  • networkTimeout 网络超时；
  • debug 是否开启debug模式；

page => index.json

细心的朋友会发现，除了app.json还有一个index.json在page文件下，在页面配置下会覆盖与全局配置相同项的配置项，页面配置.json只能配置window相关的配置项，决定本页面的窗口表现，无需写window键。

逻辑层

由于小程序框架也是由Js编写，所以在Js的基础上，小程序做了一些修改（美其名曰：为了更好地开发）

App();

App()函数是用来注册一个小程序，接收一个object参数。

getApp();

获取小程序实例。

var appInstance = getApp()
console.log(this.globalData)

page ()

page 函数用来注册页面，接受一个object参数；

事件触发，就会执行page中定义的事件处理函数；

页面的路由

初始化=>打开新页面=>页面重定向=>页面返回=>Tab切换
前三个过程路由后页面：onLoad与onShow
页面返回：只剩下onShow
触发：详见其API；

文件作用域

声名的文件与变量在同一个文件有效，不同文件可以声明同一个变量和函数；

模块化

我们可以将一些公共的代码抽离成为一个单独的 js 文件，作为一个模块。模块只有通过 module.exports 或者 exports 才能对外暴露接口。

  • 最好采用module.exports的形式将接口暴露。
  • 小程序目前不支持node_modules，
  • 需要使用模块文件时，使用require（path）将公共代码引入

    var something = require(“something.js”)
    Page({
    maybe: function() {
    something.sayHello("123")
    },
    ……
    })

条件渲染

wx：if

在框架中我们利用wx：if=“”来判断是否需要渲染改代码块
其次还有wx：elif 与 wx：else来添加else块；

使用 wx:for-item 可以指定数组当前元素的变量名，
使用 wx:for-index 可以指定数组当前下标的变量名：

事件

事件之分为：冒泡与非冒泡事件；

事件绑定

事件绑定的写法同组件的属性，以 key、value 的形式。

  • key 以bind或catch开头，然后跟上事件的类型，如bindtap, catchtouchstart
  • value 是一个字符串，需要在对应的 Page 中定义同名的函数。不然当触发事件的时候会报错。

记住：bind事件绑定不会阻止冒泡事件向上冒泡，catch事件绑定可以阻止冒泡事件向上冒泡。

事件对象

  • tyoe
  • timeStamp：事件生成时间戳
  • target：触发事件的组件的属性值集合
  • currentTarget ： 当前的属性值集合

引用的方式

  • import：一个有作用域的的引用模板的方式；
  • include：有点类似把完整模板阉割掉标签的形式传递给它引用的地方；

wxss

说白了就是css的变形记，但是相比css又拥有自己的扩展特性：

  • 尺寸单位 （用rem不好吗，hhh）：rpx可以根据屏幕自适应（1rpx=0.5px）
  • 样式的导入 ：@import可以导入外链样式表，跟需要导入的外联样式表的相对路径

    @import “common.wxss”;
    .middle-p {
    padding:15px;
    }

    内联样式

  • style：style 接收动态的样式，在运行时会进行解析，请尽量避免将静态的样式写进 style 中，以免影响渲染速度。
  • class：用于指定样式规则，其属性值是样式规则中类选择器名(样式类名)的集合，样式类名不需要带上.，样式类名之间用空格分隔。
